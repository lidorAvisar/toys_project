Toys Api

HTTP ROTE DESCRIPTION PARAMETERS/REQBODY
GET /toys Returns a list of toys according to the query parameters.
  • page: page number to return list of toys.(default 1)
    https://toys-api.cyclic.app/toys?page=1
  • perPage: numbers of toys that returns in every page.(default 5)
    https://toys-api.cyclic.app/toys?perPage=5
  • sort: returns property by sort.(default)
    https://toys-api.cyclic.app/toys?perPage=6&sort=price
  • reverse: if set to yes, sorts in ascending order.(default decending)
    https://toys-api.cyclic.app/toys?sort=price&reverse=yes
GET /toys?s= returns specific property by search.
  • searcg: search query to match specific property
    https://toys-api.cyclic.app/toys?s=Lego
GET /toys/category returns toys by category.
  • category: the name of the category to retrieve toys for.
    https://toys-api.cyclic.app/toys/category?cat=Building
GET /toys/price returns a list of toys filtered by price range.
  • min: minimum price to toy.(default 0)
    https://toys-api.cyclic.app/toys/price?min=100
  • max: maximum price to toy.(default Infinity ∞)
    https://toys-api.cyclic.app/toys/price?max=100
GET /toys/single/:id returns toys by id.
  • id: get a toy by his id.
    https://toys-api.cyclic.app/toys/single/64590ce646e0fdeb4ac6e917
POST /toys add a new toy.(token is required)
  • req.body: Toy data to be added in the following schema
  • schema: { name: String,
    info: String,
    category: String,
    img_url: String,
    price: Number}
PUT /toys/:id update toy.(token is required)
  • id: the id of the toy to update.
  • req.body: toy data to be updated in the following schema
  • schema: { name: String,
    info: String,
    category: String,
    img_url: String,
    price: Number}
DELETE /toys/:id delete toy.(token is required)
  • id: the id of the toy to delete.

Users Api

HTTP ROTE DESCRIPTION REQBODY
GET /users Test endpoint to check if the server is running N/A
GET /users/userInfo Get the information for the authenticated user (requires authentication) N/A
POST /users Create a new user { name: String, email: String, password: String }
POST /users/login Authenticate a user and receive a JWT token in response { email: String, password: String }

Users/Admin Token Abillities

Admin Default User
Has the ability to add new toys to the system Has the ability to add new toys to the system
Can edit all information related to any toy in the system Can only edit information related to toys they have created or are associated with their account
Has the ability to delete all toys from the system Can only delete toys that they have created or are associated with their account
